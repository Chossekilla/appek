Appek B2B · Instalační návod Strana 2

1. Co je Appek B2B?

Hotový balíček softwaru pro gastro provozy — všechno potřebné v jedné aplikaci.

📦 Co dostaneš

Appek B2B je kompletní krabicovka — žádný měsíční poplatek, žádný cloud, jednou koupíš a máš na celý život. Zahrnuje dvě části:

👤 Administrace pro provoz

  • 📋 Objednávky — nové, stav (potvrzená, ve výrobě, expedována), filtry
  • 🥖 Výrobní list — kolik kusů čeho péct dnes/zítra
  • 📄 Dodací listy + ✏️ Faktury (s ISDOC exportem pro POHODA/Money S3)
  • 🛣️ Rozvozové trasy — seskupené podle města pro řidiče
  • 🥖 Výrobky + kategorie + cenové skupiny
  • 📑 PDF nabídka — měsíční katalog pro odběratele
  • 🏷️ Cenovky & štítky (15 designů, formáty Avery/Printky/SEVT…)
  • 📋 HACCP dokumenty — plán, sanitační řád, postupy, audity
  • 👥 Odběratelé — kontakty + B2B přístupy
  • 🌾 Suroviny + sklad — alerty pod minimem, kalkulace
  • 🔁 Opakující se objednávky (např. „Hotel Beránek každé pondělí 30× chleba")
  • 📈 Sales report PDF — top výrobky, odběratelé, kategorie
  • 🔌 API pro účetní systémy (Money S3, POHODA, Fakturoid)

🛒 B2B portál pro odběratele

  • Vlastní login + košík + historie objednávek
  • PWA — funguje jako aplikace v mobilu
  • Push notifikace (zdarma, žádné SMS) při změně stavu objednávky
  • Šablony objednávek pro rychlé znovu-objednání
🎯
Pro koho je to vhodné?

Malé a střední gastro provozy (pekárny, cukrárny, lahůdkárny, catering, výroba), které prodávají B2B (restauracím, hotelům, jídelnám, školám). Funguje na běžném webhostingu — nepotřebuje speciální server.

2. Co potřebuješ před instalací

Příprava na 5 minut. Než začneš, ujisti se, že máš tyto věci.

📦 Soubory aplikace

✅ ZIP balíček appek-v1.0.zip
Obdržel jsi e-mailem nebo na flash disku od dodavatele. Velikost ~3 MB. Obsahuje všechny PHP soubory, CSS, JS a databázové schema.

🌐 Hosting nebo lokální server

VolbaPro kohoMěsíčně
Český webhosting
(Wedos, Hostinger, Forpsi, Active24…)		 Provoz s B2B portálem (přístupný z internetu) ~50–150 Kč
Lokální PC
(XAMPP / MAMP / Laragon)		 Testování nebo offline-only provoz 0 Kč
VPS server
(DigitalOcean, Hetzner, Forpsi VPS)		 Pokročilí, vlastní doména, plná kontrola ~200–500 Kč

📋 Server requirements (technické minimum)

PHP 8.0 nebo novější
Většina hostingů má dnes PHP 8.2+. Zkontroluj v admin panelu hostingu.
MySQL 5.7+ nebo MariaDB 10.3+
Standard u všech hostingů. Verze nemusí být nejnovější.
PHP extensions: pdo_mysql, openssl, gd, gmp, curl, mbstring, json
Standardně předinstalované na 99 % hostingů. Pokud ne, kontaktuj podporu hostingu — instalují to zdarma.
HTTPS certifikát (volitelné, doporučené)
Hostingy ho dávají zdarma přes Let's Encrypt. Bez HTTPS nepojedou push notifikace.

🔐 Co si připrav

  • FTP přístup k hostingu (uživatel, heslo, adresa) — najdeš v e-mailu od hostingového provideru po koupi
  • Email pro admin účet — např. admin@vasefirma.cz
  • Heslo (min 8 znaků) — později ho budeš používat pro přihlášení
⚠️
Domain — vlastní nebo subdoména?

Můžeš použít vlastní (vasefirma.cz) nebo subdoménu od hostingu (vasefirma.hostinger.com). Doporučujeme vlastní — vypadá to profesionálněji a stojí ~150 Kč/rok.

3. Vyber svoji cestu

Existují tři způsoby instalace. Vyber, který nejlíp odpovídá tvojí situaci.

🌐
Doporučeno

Hostinger, Wedos, Forpsi… B2B portal dostupný odkudkoliv. Strany 6–10

XAMPP/MAMP na pekárenském PC. Pro testování nebo offline-only. Strany 11–12

Vlastní server (Linux). Pro pokročilé. Strana 13

🤔 Jak se rozhodnout?

SituaceVyber cestu
Nemám hosting, ale chci ať můžou odběratelé objednávat online Cesta A
Chci si systém jen vyzkoušet, nemám hosting Cesta B
Mám malý provoz ve vesnici, slabý internet, B2B portál nepotřebuju Cesta B (lokální)
Mám už VPS, dokážu si nakonfigurovat Apache/Nginx Cesta C
Nejsem si jistý Cesta A (jistota a podpora hostingového provideru)
💡
Náklady na cestu A

Český webhosting stojí kolem 50–150 Kč/měsíc. Doména ~150 Kč/rok. SSL certifikát zdarma (Let's Encrypt). Celkem ~1 000–2 000 Kč/rok.

✅
Můžeš začít s cestou B a pak migrovat na A

Pokud chceš nejdřív vyzkoušet zdarma na lokálním PC, není problém. Později jednoduše stáhneš zálohu DB a nahráš na webhosting. Žádná data se neztratí.

4. Cesta A: Web hosting

Pro Hostinger, Wedos, Forpsi, Active24, Endora, Webglobe a jakýkoliv český hosting s PHP+MySQL.

1

Stáhni ZIP balíček

Obdržel jsi appek-v1.0.zip e-mailem nebo na flash disku. Pokud máš jen e-mail s odkazem, klikni na něj a soubor se stáhne (3 MB). Zkontroluj že je v Stažené soubory složce.

2

Vytvoř databázi v hostingovém panelu

Otevři kontrolní panel hostingu (cPanel / klientská zóna):

  • Hostinger: hPanel → Databáze → MySQL Databáze → „Vytvořit novou"

  • Forpsi: Webhosting → Databáze MySQL → „Nová databáze"
  • Active24: Administrace → Databáze → „Vytvořit"

Při vytváření si poznamenej tyto údaje (budou potřeba):

📛 Název databáze
např. u123456_appek (Hostinger automaticky přidá prefix)

např. u123456_admin (často stejný jako název)

Auto-generované hostingem nebo si zvol vlastní silné heslo
🌐 Host (server)
Většinou localhost. Některé hostingy mají vlastní server (např. mysql.wedos.net) — najdeš v dokumentaci hostingu
💾
Zapiš si přístupové údaje na papír

Tyto údaje budeš potřebovat v kroku 5. Zapiš si je na papír nebo do textového souboru. Hostingy je obvykle ukazují jen jednou.

4. Cesta A — pokračování

3

Nahraj soubory na hosting

Máš dvě možnosti, jak nahrát soubory:

🚀 Možnost A: FileZilla / WinSCP (FTP)

  1. Stáhni si zdarma FileZilla
  2. Spusť a vyplň přihlašovací údaje FTP (najdeš v e-mailu od hostingu):
    • Host: ftp.tvojefirma.cz
    • Uživatel: tvůj FTP účet
    • Heslo: tvoje FTP heslo
    • Port: 21 (nebo 22 pro SFTP)
  3. Po připojení přejdi do složky public_html/ nebo www/
  4. Rozbal ZIP lokálně na svém PC (klik pravým → Rozbalit) a celý obsah nahraj do public_html/

🚀 Možnost B: Web File Manager (přes hosting panel)

  1. V hosting panelu otevři Správce souborů
  2. Klikni „Nahrát" a vyber appek-v1.0.zip
  3. Po nahrání klikni pravým tlačítkem na soubor → „Extrahovat zde"
  4. ZIP soubor po rozbalení smaž
4

Ověř, že soubory jsou na správném místě

Otevři v prohlížeči adresu tvého webu (např. https://tvojefirma.cz/). Měl bys vidět buď:

  • ✅ Login obrazovku Appek B2B (krásná, se zlatým logo „A") — pokud ano, soubory jsou OK, jen ještě nemáš databázi
  • ⚠️ Chybu nebo seznam souborů — zkontroluj že soubory jsou v public_html/ (ne public_html/appek/)
📁
Pozor na strukturu složek

Soubory musí být přímo v public_html/, ne v podsložce. Pokud máš public_html/appek/index.html, přesuneš obsah o úroveň výš. Nebo jdi na adresu tvojefirma.cz/appek/.

5. Cesta B: Lokální XAMPP / MAMP

Bezplatný způsob, jak rozjet systém na tvém PC. Ideální pro testování, školení nebo offline-only provoz.

💻 Co je XAMPP / MAMP?

Bezplatný balíček, který nainstaluje na tvůj počítač Apache web server + PHP + MySQL. Zachová se jako mini-hosting přímo na tvém PC. Funguje na Windows, macOS i Linuxu.

🪟

Windows: XAMPP

apachefriends.org
~150 MB
🍎

macOS: MAMP

mamp.info
~500 MB
🐧

Linux: Apache + PHP

apt install apache2 php mysql
1

Nainstaluj XAMPP nebo MAMP

Stáhni z oficiální stránky (odkazy výše) a nainstaluj jako jakýkoliv jiný program. Použij výchozí nastavení.

2

Spusť server

Otevři XAMPP / MAMP Control Panel a klikni Start u Apache i MySQL. Měli by zezelenat.

Otevři v prohlížeči http://localhost — měl bys vidět XAMPP/MAMP uvítací stránku.

3

Rozbal ZIP do htdocs

Najdi složku htdocs/ (XAMPP) nebo MAMP/htdocs/ (MAMP). Tam rozbal ZIP balíček.

  • Windows XAMPP: C:\xampp\htdocs\
  • macOS MAMP: /Applications/MAMP/htdocs/

Vytvoř podsložku appek/ a rozbal tam soubory.

4

Vytvoř databázi v phpMyAdmin

Otevři http://localhost/phpmyadmin. Klikni Nová a vytvoř databázi:

  • Název: appek
  • Collation: utf8mb4_unicode_ci

🌐 Použití B2B portálu z mobilu / jiného PC

Pokud chceš, aby B2B portál (odběratelé) byl dostupný i z mobilu nebo jiných PC v síti:

  1. Zjisti IP adresu tvého PC (Windows: ipconfig, Mac: ifconfig)
  2. Z mobilu na stejné Wi-Fi otevři http://192.168.1.X/appek/
  3. (Volitelné) Otevři port 80 ve firewallu PC
⚠️
Lokální = bez veřejné dostupnosti

B2B portál bude dostupný jen z tvojí lokální sítě. Odběratelé nebudou moci objednávat z domova přes internet. Pokud to chceš, použij cestu A.

🔄
Auto-start XAMPP při startu PC

Aby se XAMPP spouštěl automaticky při zapnutí PC: XAMPP Control Panel → Config → Service and Port Settings → ✓ Apache + MySQL → Save.

💾
Migrace lokálně → cloud později

Až budeš chtít přejít na webhosting (cesta A): V adminu klikni Zálohy databáze → Stáhnout, pak na novém hostingu po instalaci nahraj zálohu přes phpMyAdmin. Všechna data se přesunou.

6. Cesta C: VPS server

Pro pokročilé. Vlastní Linux server (DigitalOcean, Hetzner, Forpsi VPS…). Plná kontrola, vlastní doména, větší výkon.

🚀 Předpoklady

  • VPS s Ubuntu 22.04+ nebo Debian 12+
  • SSH přístup (root nebo sudo user)
  • Vlastní doména s A záznamem na VPS IP

📦 Instalace LAMP stack

# Update + instalace Apache, PHP, MySQL
sudo apt update && sudo apt upgrade -y
sudo apt install -y apache2 mysql-server \
  php php-mysql php-curl php-gd php-gmp \
  php-mbstring php-xml php-zip libapache2-mod-php

# Zabezpečení MySQL (interaktivní)
sudo mysql_secure_installation

# HTTPS přes Let's Encrypt
sudo apt install -y certbot python3-certbot-apache
sudo certbot --apache -d tvojefirma.cz

📂 Nahrání souborů

# Z lokálního PC:
scp appek-v1.0.zip root@vps-ip:/var/www/

# Na VPS:
cd /var/www/
unzip appek-v1.0.zip -d html/
chown -R www-data:www-data /var/www/html/
chmod -R 755 /var/www/html/

🗄️ Databáze

sudo mysql -u root -p
CREATE DATABASE appek CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'appek'@'localhost' IDENTIFIED BY 'silne_heslo_zde';
GRANT ALL PRIVILEGES ON appek.* TO 'appek'@'localhost';
FLUSH PRIVILEGES;
EXIT;

🔧 Apache config

# Aktivuj rewrite + headers moduly
sudo a2enmod rewrite headers expires deflate
sudo systemctl restart apache2

# .htaccess je už součástí balíčku
# Nginx config je v souboru nginx.conf.example

🌐 Spuštění instalátoru

Otevři v prohlížeči https://tvojefirma.cz/install.php a postupuj stejně jako v cestě A (kroky 5-8).

⚡ Cron pro opakující se objednávky

sudo crontab -e
# Přidej řádek:
0 4 * * * php /var/www/html/api/cron_recurring.php
📖
Nginx místo Apache?

V balíčku je nginx.conf.example s kompletní konfigurací včetně PHP-FPM, security headers a HTTPS. Zkopíruj do /etc/nginx/sites-available/.

7. První přihlášení + Onboarding

Po instalaci tě uvítá onboarding wizard. Provede tě 6 kroky a připraví aplikaci na reálné používání.

🚪 První přihlášení

  1. Otevři /admin/ v prohlížeči
  2. Vyplň e-mail a heslo (z kroku 7 instalace)
  3. Klikni Přihlásit

🎯 Onboarding wizard — 6 kroků

1

🏢 Údaje firmy

Zadej IČO a klikni 🔍 Načíst — systém automaticky stáhne data z ARES (CZ) nebo RPO (SK): název, adresa, DIČ. Můžeš ručně upravit.

2

🖼️ Logo

Nahraj logo firmy (PNG/JPG, doporučeno 400×400). Systém automaticky vygeneruje i favicon (32×32 ikona v záložce prohlížeče).

3

🏷️ Kategorie výrobků

Vyber které kategorie budeš mít (Chleby, Pečivo, Koláče, Zákusky…). Můžeš si nechat naplnit i 15 vzorových výrobků abys měl rychlý start.

4

👥 Prvni odběratelé

Můžeš naimportovat ze starého systému (CSV/JSON), z vizitek (vCard), nebo začít prázdně a přidávat ručně.

5

📋 HACCP dokumenty

Systém automaticky vygeneruje sadu HACCP dokumentů personalizovaných daty tvé firmy (z kroku 1). Plán HACCP, Sanitační řád, Postupy CCP, Záznamy o auditech.

6

✅ Hotovo

Onboarding dokončen. Můžeš začít vytvářet objednávky, faktury, cenovky atd.

💡
Onboarding přeskočit?

Ano, klikni Přeskočit v jakémkoliv kroku. Všechno najdeš později v Nastavení. Onboarding můžeš spustit znovu konzolí: restartOnboarding().

8. Zálohy databáze

Nejdůležitější věc po instalaci. Bez záloh = riziko ztráty dat. Settings → Údržba → Zálohy.

💾 Co se zálohuje

  • Celá MySQL databáze (~30 tabulek): objednávky, faktury, odběratelé, výrobky, HACCP, nastavení
  • Volitelně: složka uploads/ (loga, obrázky výrobků, podpisy odběratelů)

📦 Tři způsoby zálohování

1️⃣ Ruční záloha (1× za týden)

V adminu: Nastavení → Údržba → Zálohy databáze → 💾 Zálohu DB. Soubor se vytvoří na serveru, stáhneš si ho do PC.

2️⃣ Automatická záloha přes cron (denně v noci)

Pokud hosting podporuje cron, nastav úlohu:

0 3 * * * php /cesta/k/appek/api/cron_zaloha.php

Pojede automaticky každý den ve 3:00. Záloha se uloží do api/zalohy/.

3️⃣ Manuálně přes phpMyAdmin

Otevři phpMyAdmin → vyber databázi → Export → SQL. Stáhne se ti .sql soubor.

⚠️
Doporučujeme 3-2-1 pravidlo

3 kopie dat · 2 různá média · 1 mimo budovu provozu. Příklad: záloha na serveru + záloha v PC + záloha v cloudu (Google Drive / Dropbox).

🔄 Obnova ze zálohy

Pokud potřebuješ obnovit databázi (např. po hackeru nebo chybě):

  1. V adminu: Zálohy → najdi zálohu → ↺ Obnovit. Dvojí potvrzení (bezpečnost).
  2. Přes phpMyAdmin: vymaž databázi → Import → vyber SQL soubor → Importovat.
  3. Přes SSH: mysql -u root -p appek < zaloha.sql
✅
Před každou velkou změnou si udělej zálohu

Než budeš měnit DPH sazby, mazat odběratele nebo upravovat schema — vždy nejdřív klikni 💾 Zálohu DB. Trvá to 5 vteřin a může ti zachránit hodiny práce.

9. Časté problémy a řešení

Najdi svůj problém v tabulce a vyřeš za 30 vteřin.

🚨 Při instalaci

ProblémŘešení
Po otevření adresy vidím seznam souborů Server nemá zapnutý mod_rewrite nebo DirectoryIndex. Kontaktuj hosting, požádej o zapnutí. Nebo otevři přímo /index.html.
Install.php vypisuje „Internal Server Error 500" Některá PHP extension chybí. Otevři Diagnostiku po instalaci. Většinou chybí gd nebo gmp — hosting to zapne na požádání.
DB test selhává: „Access denied" Špatné heslo nebo uživatel. Vrať se do hosting panelu a zkontroluj přístupy. Pozor — u Hostingeru jsou DB hesla auto-generovaná a obsahují speciální znaky.
DB test selhává: „Unknown database" Databáze ještě nebyla vytvořena. Vrať se do hosting panelu → MySQL → Vytvořit. Pozor na prefix (Hostinger: u123456_).
DB test selhává: „Connection refused" / „Can't connect" Špatný host. Standardně localhost, ale některé hostingy mají vlastní (Wedos: mysql.wedos.net). Najdi v dokumentaci hostingu.

🐛 Po instalaci

ProblémŘešení
Emaily se neodesílají Hosting blokuje PHP mail(). V Nastavení → Notifikace zapni SMTP nebo požádej hosting o aktivaci mail funkce. Otestuj v Diagnostice.
Logo se nenahraje Složka uploads/ nemá zápisová oprávnění. Přes FTP nastav permissions 755 nebo 775 pro celou složku uploads/.
Push notifikace nefungují Vyžadují HTTPS. Pokud máš HTTP, push není dostupný. Nainstaluj SSL certifikát (zdarma přes Let's Encrypt v hostingu).
Aplikace je pomalá Otevři Nastavení → Údržba → Diagnostika. Tam se ukáže, jestli chybí MySQL indexy nebo je DB příliš velká. Klikni 🔬 Lint API.
Faktura PDF se nevygeneruje Faktury jsou HTML+CSS, ne PDF library. V prohlížeči klikni Tisk → Uložit jako PDF. Funguje ve všech prohlížečích.

9. Časté problémy — pokračování

🌐 B2B portál

ProblémŘešení
Odběratel se nemůže přihlásit Přihlásí se na admin, pak Odběratelé → najdi účet → ✏️ Upravit → záložka „Přístup" → změň heslo. Email vždy malými písmeny.
Odběratel nevidí žádné výrobky Buď nemá přiřazenou cenovou skupinu, nebo výrobky nejsou aktivní. Zkontroluj v Odběratelé → cenová skupina + Výrobky → aktivní.
B2B se neotevírá v mobilu jako aplikace Pouze HTTPS web může být PWA. Pokud máš HTTP, B2B funguje jako normální web. Po přechodu na HTTPS klikni v Chrome menu → Přidat na plochu.

🔧 Server

ProblémŘešení
Hláška „PHP 8.0+ vyžadováno" V hosting panelu změň PHP verzi na 8.0 nebo vyšší. U Hostingeru: hPanel → Pokročilé → PHP verze.
Chybí PHP extension gmp Push notifikace nepojedou bez GMP. U většiny hostingů: kontaktuj podporu, zapnou ti GMP zdarma. Ostatní funkce fungují bez GMP.
Cron nefunguje Hosting nemá cron. Použij web-based cron: v Nastavení vygeneruj cron_token a volej URL /api/cron_recurring.php?token=... z externího cron služby (cron-job.org, EasyCron, IFTTT).
Aplikace občas spadne s „500 Internal Server Error" Většinou nedostatek PHP paměti. Vytvoř .user.ini v rootu s obsahem memory_limit = 256M. Pokud to nepomůže, hosting nepomůže — přejdi na výkonnější tarif.

📞 Když nic nezabírá

Otevři Nastavení → Údržba → 🩺 Diagnostika → Otevřít detail. Tam je kompletní přehled systému (PHP verze, extensions, DB stav, schema integrita). Pošli screenshot dodavateli, případně hostingovému administrátorovi.

10. Bezpečnostní checklist

Po instalaci projeď tento checklist. Trvá 15 minut, ale ušetří ti ti spoustu starostí.

🔒 Nezbytné (proveď do 24 hodin)

  • ☐ Smazat install.php — útočník by mohl resetovat databázi
  • ☐ Silné admin heslo (12+ znaků, čísla, písmena, speciální)
  • ☐ HTTPS aktivovaný — bezplatný Let's Encrypt v hostingu
  • ☐ Denní zálohy zapnuté (str. 15)
  • ☐ Vyplnit ARES údaje firmy — abys neměl podezřele prázdné PDF faktury

🛡️ Doporučené (do 1 týdne)

  • ☐ Dvojité potvrzení mazání zapnuto (Nastavení → Údržba → Bezpečnost)
  • ☐ Cron nastaven pro opakující se objednávky
  • ☐ Vytvořit roli „prodavač" s omezeným přístupem (žádné mazání)
  • ☐ API tokens pro účetní — vygenerovat jeden token na účetního, nikoliv sdílet admin heslo
  • ☐ Smazat ne-potřebné soubory: build-zip.sh, *.example, README.md (volitelné, ale čistší)

🔐 Pokročilé (volitelné)

  • ☐ IP whitelist pro admin v .htaccess: 
    <Files "admin/index.html">
  Require ip 192.168.1.0/24
</Files>
  • ☐ Změnit URL adres adminu — místo /admin/ např. /spravce/
  • ☐ Fail2ban na VPS — auto-bany IP po 5 špatných pokusech přihlášení
  • ☐ Off-site backup — záloha do externího cloudu (Google Drive, Dropbox)

🚨 Co dělat při hacku

  1. Okamžitě změnit admin heslo
  2. Změnit DB heslo v hosting panelu + v api/config.local.php
  3. Stáhnout zálohu DB ze před útokem (z api/zalohy/)
  4. Stáhnout celou aplikaci přes FTP a zkontrolovat změny souborů (porovnat s originálním ZIP)
  5. Smazat všechny soubory na hostingu a nahrát originální ZIP znovu
  6. Obnovit databázi ze čisté zálohy
  7. Změnit hesla VŠEM uživatelům (admin + odběratele)
⚠️
Veřejně dostupný systém = cíl útočníků

Internet je nebezpečný. Boti hledají pekárenské systémy a zkouší přihlásit. .htaccess v balíčku má vestavěnou ochranu, ale to nestačí — pravidelně aktualizuj a měň hesla.

11. Aktualizace systému

Když dodavatel vydá novou verzi (např. v1.1, v1.2…), takhle ji nainstaluješ bez ztráty dat.

🚀 Aktualizace v 4 krocích

1

Vytvoř zálohu

V adminu: Nastavení → Údržba → Zálohy → 📦 DB + uploads. Stáhni si zálohu do PC.

2

Stáhni novou verzi

Dodavatel ti pošle appek-v1.1.zip e-mailem nebo na flash disku.

3

Nahraj přes FTP

Rozbal ZIP a všechny soubory nahraj přes FTP do public_html/. Volba Přepsat existující. NESMĚŠ smazat tyto soubory:

  • api/config.local.php — tvoje DB credentials
  • api/.installed — flag že je nainstalováno
  • uploads/ — všechny obrázky a loga
  • api/zalohy/ — staré zálohy

FTP klient se zeptá „Soubor existuje, přepsat?" → vyber Přepsat vždy, ale ručně zaškrtni „Vynechat" pro výše uvedené soubory.

4

Otevři admin

Po nahrání otevři /admin/. Aplikace automaticky:

  • Detekuje novou verzi
  • Spustí auto-migraci databáze (přidá nové sloupce/tabulky)
  • Vyčistí cache prohlížeče (force-refresh)

🆘 Co když se update pokazí

  1. Nahraj zpět starou verzi (z předchozího ZIP)
  2. Obnov databázi ze zálohy (krok 1 update)
  3. Kontaktuj dodavatele s popisem chyby + screenshot Diagnostiky
✅
Auto-migrace = bezpečná

Appek nepoužívá DELETE migrace. Nové verze pouze přidávají sloupce/tabulky, nikdy nemažou existující data. Update je tedy vždy reversibilní.

⏰
Kdy updatovat?

Doporučujeme jednou za 3 měsíce, ne každý měsíc. Update v době mimo špičku (po obědě, nedělním večeru). Trvá 5–10 minut.
